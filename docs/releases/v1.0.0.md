# FilterNArrange v1.0.0 — release notes

Plan H §T12. The first public, tagged release.

## What v1.0.0 includes

**Core pipeline** (Plan A → B → C)
- Upload, detect, filter, convert, analyze. Eight formats out of the box
  (CSV/TSV/JSON/JSONL/XML/YAML/XLSX, plus paste-as-text).
- Four filter modes (column / row / regex / expression).
- Four analysis modes (summary-stats / group-by / chart-suggest / schema-infer).

**Async + Kafka** (Plan D)
- Job queue with idempotency, heartbeat, and WebSocket live progress.
- Tier-routed topics (`topic.v1.jobs.paid` / `topic.v1.jobs.free`).
- 503 fallback on data-engine breaker via the dataEngine circuit-breaker.

**AI** (Plan E)
- Four OSS LLM capabilities over Ollama: NL-to-filter, auto-summary,
  chart-suggest, anomaly-detect.
- Redis-cached responses, semaphore-bounded concurrency (default 4).

**Tiers + features** (Plan F)
- Subscriptions with one-active-row-per-user invariant.
- Daily op quota, upload size cap, feature-gate, IP rate limit — all
  fail-open on Redis outage.
- Recipes (paid), format-requests with GitHub-issue mirror (paid),
  billing endpoint, admin triage page.
- Python retention worker sweeps aged uploads / results per tier policy.

**Hardening** (Plan G)
- Keycloak compose service + V10 users.external_id migration (full OIDC
  rollout in the Plan G PR-1 follow-ups).
- Prometheus + Grafana + AlertManager + node-exporter + SLO alerts.
- Nightly pg_dump → MinIO with 14-day retention; mc-mirror of blobs.
- Migration policy doc + four incident runbooks.

**Public deploy** (Plan H — this release)
- Production compose overlay sized for Oracle Always-Free A1 24 GB.
- Caddy + ACME with rate-limit module and full security headers.
- Host bootstrap script + systemd unit + logrotate.
- Smoke test suite for post-deploy verification.
- Provisioning runbook, secrets policy, post-launch monitoring.

## Known limitations

- **Single-VM, single-region.** v1.0 is one Oracle A1 host. A complete
  host loss requires manual restore from the most recent backup.
- **Spring-jwt is the default auth.** Keycloak service is provisioned
  but the AuthFilter switchover (Plan G T3-T6) ships in a follow-up.
- **No PITR.** Backups are nightly pg_dumps; data loss window is up to 24h.
- **No off-VM backup.** Plan H next-up: Backblaze B2 mirror.

## Tag procedure

```sh
git checkout main
git pull --ff-only
git tag -a v1.0.0 -m "FilterNArrange v1.0.0 — public deploy"
git push origin v1.0.0
```

The tag-triggered deploy workflow (Plan H §T7 follow-up) builds the
multi-arch images, signs them with Cosign, pushes to GHCR, and rolls
the VM.

## Upgrading

There's nothing to upgrade FROM — v1.0.0 is the first tagged release.
Future versions: bump ``FNA_VERSION`` in ``/etc/filternarrange/prod.env``
and ``systemctl restart filternarrange``.
